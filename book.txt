# С генераторами легко попасть в ловушку, если пытаться делать всё только с
# помощью функций. В результате может получиться сложный код, если
# генератору нужно взаимодейст python вовать с другими частями программы некими
# необычнымыми способами (раскрытие атрибутов, разрешение на управление
через вызов методов и т.п.) В этом случае просто используйте определение
класса, как показано выше. Определение генератора в методе __iter__() не
#изменит ничего в том, как python вы напишете алгоритм. Но тот факт, что генератор
# станет частью класса, упростит задачу предоставления пользователям
атрибутов и методов для каких-то взаимодействий.
#Потенциальная хрупкость python показанного приёма заключается в том, что он
может потребовать дополнительного шага: вызова iter(), если вы собираетесь
